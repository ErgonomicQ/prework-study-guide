HTML

  • The head element contain information about the webpage.
  • The body element represents the visible content shown to the user.
  • The Title element defines a webpages title. This Element is required for publishing and helps SEO.
  • The meta element defines metadata such as character set, descriptions, author, et cetera. Typically this is not seen by users.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how mcuh space we want around the content inside of an element.
  • Style attibute can be use to change a single h1 heading. Less flexible as the changes are individual to the line.
  • Style Element is internal and the rules for the PAGE itself in the Head element.
  • Commonly though, a link element is used for an external CSS style sheet and stored as a separate file, where changes are easier to make.
  • Padding, while transparent, is located inside the element, whereas the margin is the transparent buffer outside an element.
  • It is important to know that every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content. This is sometimes referred to as the CSS box model.
  • The period preceding a "class" designates this as a class selector. Any HTML element assigned to the class name will gain these property assignments from the CSS

Git

  • "git" is the default command while working in a github workspace.
  • .. status - checks up to date status on branch.
  • .. checkout -b branch-name - creates and swaps to a new branch off of the main called branch-name.
  • .. add -A - adds all changes to the branch.
  • .. commit -m - commits all changes with a message text.
  • .. pull origin main - pull from source to a local enviroment/repo/branch we're pulling from.
  • .. push origin branch-name - push te local changes to the repo, creates a remote branch
